CSS Files Structure

Tomorrow Water uses a number of CSS and SCSS files in order to keep the template organized and easy to use for further development. Knowing CSS files makes customization process a lot easier and far more flexible. The

CSS

  • ---- pages: From here you may control all of the independent CSS pages.
  • ---- headers: From here you may control all of the 8 Headers. They are in a separate folder because usually one header is used per project.
  • ---- footers: From here you may control all of the 8 Footers. They are in a separate folder because usually one footer is used per project.

Blog System

The blog system utilizes Tumblr API to generate and output posts to Tomorrow Water, simplifying the work process for BKT to create, modify, and update blog contents.

Here is the step by step guideline to create blogs

  • Go to tumblr.com and access the Tomorrow Water account
  • Click on ‘make a post’ button and insert title and contents
  • The post must contain one of the following tags. The tags will be used to identify where the post will be displayed among Press, Technologies, and Partnership sections under Media.
    • tw-press (post will be displayed in Press)
    • tw-technologies (post will be displayed in Technologies)
    • tw-partnerships (post will be displayed in Partnership)
  • Under the technology section, if the post is technology specific (e.g. BBF, AAD, etc), adding the technology name as its tag in addition to tw-technologies tag allows the blog client to recognize the post and filter posts by technology. You may add multiple tags as needed
    • For example, if a post is about EOS, please add EOS as tag.
    • If a post is about BBF and AAD, add both BBF and AAD as tags.

Deployment

The developer environment is hosted on a github remote repository which stores all data for Tomorrow Water project. GitHub is a cloud service that uses git VCS (Version Control System) to manage workflow and data backups in software development.

Currently, the repository contains a single master branch to manage the entire development process. The updated files must be pushed to display the website with all updates. Once the repository is up to date, the website will be automatically deployed with latest changes. Please refer to git documentation on getting started with the version control system.

End of Documentation
